# SKILL.md 编写最佳实践

## 1. description 要精准

- 列举典型触发词
- 明确使用场景
- 避免模糊描述

**❌ 差的描述：**
```yaml
description: 用于写文章时使用
```

**✅ 好的描述：**
```yaml
description: 当用户提到'公众号文章'、'自媒体内容'、'写一篇推文'、'发布到公众号'等关键词，或者要求将文章发布到微信时触发此技能
```

## 2. 工作流要"傻瓜化"

- 步骤清晰、可执行、无歧义
- 不要假设 AI 会"举一反三"
- 每步说明输入和输出
- 像给新人写 SOP 一样清晰

## 3. 保持主文件简洁

- SKILL.md 主文件建议控制在 500 行内
- 复杂内容拆分到 `references/` 子文件中
- 采用"渐进式披露"原则

## 4. 明确工具调用规范

- 说明参数格式和返回值
- 定义错误处理策略
- 设置重试和降级机制

## 5. 提供充分示例

- 输入输出示例帮助 AI 理解期望
- 边界案例说明避免意外行为
- 模板文件确保输出一致性

## 6. 合理组织资源文件

- `references/` 存放规范文档、检查清单
- `assets/` 存放模板文件、示例文件
- `scripts/` 存放可执行脚本
